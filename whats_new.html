v1.5의 새로운 기능

이 단락에서는 이번 릴리즈에 도입된 주요 기능을 찾으실 수 있습니다. 보든 변경점을 보시려면 번들러 1.5의 changelog에 문서화되어 있습니다. 1.5의 전체 changelog

--jobs (줄여서 -j)옵션으로 페러럴 인스톨이 가능합니다. 예를들어, bundle install -j4는 4개의 워커를 사용합니다. 새로 번들 인스톨을 할경우 40-60%정도의 속도 향상이 있었습니다. 항상 페러럴하게 인스톨 하려면 bundle config --global jobs 4 를 실행하거나 BUNDLE_JOBS를 설정하세요.
자세한 내용은: Parallel Install

번들러는 이제 bundle config로 Gemfile에 지역적으로 미러를 사용할 수 있게 되었습니다. bundle config mirror.http://rubygems.org http://rubygems-mirror.org/
자세한 내용은: Source Mirrors

DSL ruby 에서 이제 특정 패치레벨을 찾기위해 :patchlevel 옵션을 사용할 수 있습니다. ruby '2.0.0', :patchlevel => "247"
자세한 내용은: Ruby Directive

bundle outdated --strict 의존성을 충족하는 새로운 버전이 나온 젬들을 표시합니다. displays outdated gems that match the dependency requirements.
자세한 내용은: outdated --strict

bundle install 에서 이제 실패한 다운로드를 재시도할 수 있습니다. 재시도 횟 수를 --retry 옵션으로 조정하세요.
자세한 내용은: Retry

번들러 1.5에는 다음 패치들도 포함됩니다.
  • resolve와 인스톨을 빠르게 하기위한 많은 작은 성능개선
  • 무한 루프 방지용, 순환참조 감지
  • bundle binstubs 명령어에 다중 인자
  • bundle에서 오타 냈을 경우를 위한 bundler 명령어
  • 윈도우즈와의 더 좋은 호환성을 위한 RUBYLIB
    • 1.5의 전체 changelog
